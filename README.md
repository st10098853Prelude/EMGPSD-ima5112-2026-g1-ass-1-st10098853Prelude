# Project Overview

The **SparkSnack Social App** is a mobile application developed as part of an assignment in the IMAD5112 subject. This application was created using **Kotlin** and **Android Studio**.

The main purpose of the app is to help users maintain social connections throughout the day by suggesting small social actions based on the time of day entered by the user.

The app encourages users to stay connected with friends, family, and colleagues by providing simple and fun interaction suggestions.

The project also demonstrates the use of **GitHub for version control** and **GitHub Actions for automated builds**.

---

# Purpose and Features

## Purpose
The main goal of this app is to encourage users to maintain social connections during busy days by providing quick and simple social interaction suggestions based on the time of day.

## Key Features

- **Time of Day Input**  
  Users can type a time of day such as morning, afternoon, or dinner.

- **Social Spark Suggestions**  
  The app suggests a small social interaction based on the entered time.

- **Reset Button**  
  Allows the user to clear the input field and suggestion results.

- **Error Handling**  
  If the user enters an invalid time of day, the app displays a helpful message asking them to try again.

- **Simple User Interface**  
  The app has a clean and easy-to-use design.

These features aim to provide a quick and engaging way for users to stay socially connected.

# Design Considerations

The design of the **SparkSnack Social App** was based on the following considerations:

# GitHub and GitHub Actions

This project was managed using **GitHub** for version control. Code was regularly committed and pushed to the repository to keep track of changes and maintain a backup of the project.

GitHub helps maintain the project history and ensures that the development process is properly documented.

## GitHub Actions

GitHub Actions was used to automate the building process of the Android application. The workflow automatically:

- Builds the Android project
- Compiles the app into APK format
- Uploads the build artifacts

This automation ensures that the app can be built successfully every time changes are pushed to the repository.

# Challenges and Learnings

During the development of this project, several challenges were encountered.

### Challenge 1: Handling User Input
Users could enter different variations of time such as uppercase or lowercase.

**Solution:**  
Input was converted to lowercase before checking conditions to ensure the app recognizes valid inputs.

### Challenge 2: Implementing Conditional Logic
The app required different suggestions based on specific times of day.

**Solution:**  
Kotlin **if statements** were used to evaluate the user input and display the appropriate suggestion.

Through this project, I gained experience in:

- Android app development using Kotlin
- Designing Android user interfaces with XML
- Using GitHub for version control
- Implementing conditional logic in mobile applications
